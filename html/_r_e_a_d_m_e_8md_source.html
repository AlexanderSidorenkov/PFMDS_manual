11 ## Что это?
12 
13 Это набор программ для молекулярной динамики (МД), подгона потенциалов и анализа результатов.
14 
15 Поддерживается OpenMP параллелизм. Несколько независимых прогонов МД могут быть запущены параллельно с помощью MPI.

27 ### run_md_simulation и run_md_simulation_mpi
28 
29 Моделирование методом МД.
30 
31 Потенциалы:
32 
33 * Леннарда-Джонса (между двумя группами атомов и внутри одной) - lj, lj1g
34 * модифицированный Леннарда-Джонса c косинусом (графен - металл) - ljc
35 * модифицированный Морзе c косинусом (графен - металл) - morsec
36 * Розато-Жиллопа-Легранда для металлов (один тип атомов) - rjl
37 * Терсоффа-Бреннера (углерод) - tb
38 * REBO (углерод, только подсчет энергии, сил нет) - rebosc
39 
40 Интеграторы:
41 
42 * Верле в скоростной форме - nve
43 * Верле в скоростной форме с цепочкой термостатов Нозе-Гувера (пока возможно применение только одной цепочки) - nvt
44 * Молекулярная статика - nvms
45

47 
48 Подгон параметров модифицированных потенциалов Леннарда-Джонса или Морзе.
49 
50 ### run_gr_analysis
51

59

79 ## Как запуситить моделирование?
80 
81 Исполняемые файлы в папке executables - результат компиляции программ в папке code_source/runnes, в которых вызываются необходимые подпрограммы. Для запуска моделирования достаточно использовать программу run_md_simulation или run_md_simulation_mpi.
82 
83 ### Запуск run_md_simulation:
84 
85 Запускает программу МД на одном процессоре (вычислительном узле). Если указан список файлов настроек, то будет по очереди запущено несколько прогонов.
86 
87 Опции run_md_simulation. В скобках указаны значения по умолчанию.
88 
89 -i,--input (md_run_settings.txt) Имя файла настроек.
90 
91 -p,--prefix () Префикс выходных файлов.
92 
93 -op,--out_period (1) Период вывода в шагах МД.
94 
95 -omp_n,--openmp_threads_num (максимальное количество потоков) Количество OpenMP потоков.
96 
97 -ipath,--input_path () Путь к папке с входными файлами
98 
99 -ilist,--input_list () Имя файла со списком файлов настроек.
100 
101 -opath,--out_path () Путь к папке с выходными файлами.
102 
103 -ofile,--all_out_file (all_out.txt) Имя файла для общего вывода нескольких прогонов.
104 
105 ### Запуск run_md_simulation_mpi:
106 
107 Запускает программу МД на нескольких процессорах (вычислительных узлах). Если указан список файлов настроек, то несколько прогонов будут распределены между доступными узлами (количество прогонов должно быть равно или больше количества вычислительных узлов). Если список не указан, то один прогон будет независимо запущен на всех узлах.
108 
109 run_md_simulation_mpi имеет те же опции, что и run_md_simulation
110 
111 ### Входные файлы
112 
113 #### Файл настроек
114 
115 Имеет фиксированный порядок строк и переменных в строках, который определяется порядком чтения в md().
116 
117 Названия параметров в файле настроек не проверяются при чтении. Единственное что важно, это порядок!
118 
119 Пример:
120 
121 Двухкомпонентный Л-Дж газ из атомов A и B.
122 
123 Сначала проводится моделирование при постоянной температуре, потом при постоянной энергии, потом все охаждается до 0 К молекулярной статикой.
124 
125 ```
126 md_step_limit: 1000000
127 logfilename: md_run.log
128 init_xyz_filename: AB.xyz
129 new_velocities: T
130 zero_momentum_period: 1000000
131 particle_types_num: 2
132 groups_num: 4
133  1 A B
134  2 A #
135  3 B #
136  4 # #
137 all_moving_atoms_group_num: 1
138 xyz_moving_atoms_group_num: 1
139 z_moving_atoms_group_num: 4
140 termo_atoms_group_num: 1
141 all_atoms_group_num: 1
142 traj_group_num: 3
143 period_traj: 200
144 change_group_num: 0
145 invert_z_vel: F
146 integrators_num: 3
147  name dt len snap log
148  nvt 0.50000 10000 500000 1000
149  nve 0.50000 10000 500000 1000
150  nvms 0.50000 10000 500000 1000
151 ms_de: 1.e-8
152 nhc_params: 100. 3 10000.
153 initial_temerature: 100.
154 interactions_num: 3
155 lj parameters_LJ_A-B.txt
156 2 3 8 7.5 20
157 3 2 64 7.5 20
158 lj1g parameters_LJ_A-A.txt
159 2 2 64 7.5 20
160 lj1g parameters_LJ_B-B.txt
161 3 3 8 7.5 20
162 ```
163 
164 md_step_limit - лимит шагов МД.
165 
166 logfilename - имя файла для регулярного вывода энергий и температуры.
167 
168 init_xyz_filename - имя .xyz файла с начальными расположениями и скоростями атомов.
169 
170 new_velocities - T или F, присваивать ли атомам новые скорости. Если T, то атомы из группы all_moving_atoms_group_num будут иметь скорости, распределенные по Максвеллу для температуры initial_temerature.
171 
172 zero_momentum_period - период обнуления скорости центра масс группы атомов all_atoms_group_num. При долгих временах моделирования могут накопиться ошибки. Угловые скорости не обнуляются.
173 
174 particle_types_num - количество рассматриваемых типов атомов.
175 
176 groups_num - количество групп атомов. Один атом может входить в несколько групп или не входить ни в одну. Скорее всего понадобится пустая группа.
177 
178 После groups_num идет таблица с groups_num строками и particle_types_num+1 столбцами. Первый столбец - вспомогательный, рекомендуется писать в него номера строк, они же номера групп. Остальные столбцы - имена атомов.
179 
180 all_moving_atoms_group_num - номер группы со всеми двигающимися атомами.
181 
182 xyz_moving_atoms_group_num - номер группы с атомами, двигающимися во всех направлениях
183 
184 z_moving_atoms_group_num - номер группы с атомами, двигающимися только по оси Z. Группы xyz_moving_atoms_group_num и z_moving_atoms_group_num не должны пересекаться! Термостат не учитывает уменьшение количества степеней свободы для этих атомов!
185 
186 termo_atoms_group_num - термостатируемая группа атомов.
187 
188 all_atoms_group_num - группа со всеми моделируемыми атомами.
189 
190 traj_group_num - группа атомов для вывода положений атомов в один файл каждые period_traj шагов МД.
191 
192 period_traj - период вывода положений атомов из группы traj_group_num.
193 
194 change_group_num - количество групп с изменяемым количеством атомов (для добавления атомов в ячейку). См. ???????.
195 
196 invert_z_vel - T или F, создать или нет абсолютно упругую стенку в верхней части ячейки (для предотвращения прохода атомов через Z границу ячейки)
197 
198 integrators_num - количество интеграторов.
199 
200 Далее идет вспомогательная строка.
201 
202 Далее integrators_num строк с названием интегратора, шагом интегрирования, количеством шагов, периодом вывода положений и скоростей атомов в отдельный файл и периодом вывода в logfilename.
203 
204 ms_de - разница энергии для остановки молекулярной статики.
205 
206 nhc_params - параметры цепочки термостатов Нозе-Гувера: температура, количество звеньев и масса первого звена.
207 
208 initial_temerature - начальная температура группы all_moving_atoms_group_num если в new_velocities указано T.
209 
210 interactions_num - количество взаимодействий между атомами.
211 
212 Далее идут interactions_num блоков с параметрами для каждого взаимодействия: название взаимодействия, название файла с параметрами потенциала; в селующих строках - параметры списков соседей.
213 
214 Список соседей имеет следующие параметры (по порядку): номер первой группы атомов, второй группы атомов, максимальное количество соседей из второй группы для атомов из первой, максимальный радиус включения в список соседей (должен быть меньше радиуса обрезания потенциала и как минимум в два раза меньше минимального размера ячейки), период обновления списка в шагах.
215 
216 #### Файл .xyz с начальными положениями и скоростями атомов.
217 
218 Его имя указывается в файле настроек (init_xyz_filename). Также содержит размеры вычислительной ячейки, массы атомов и их названия.
219 
220 Пример:
221 
222 4 атома A и 4 атома B с нулевыми скоростями в ячейке 16*16*16
223 ```
224  8
225 Lattice=" 16.0 0.0 0.0 0.0 16.0 0.0 0.0 0.0 16.0 " Properties=pos:R:3:vel:R:3:mass:R:1:species:S:1
226  6.0000000 6.0000000 6.0000000 0.0000000 0.0000000 0.0000000 10.0000000 B
227  6.0000000 6.0000000 10.0000000 0.0000000 0.0000000 0.0000000 10.0000000 B
228  6.0000000 10.0000000 6.0000000 0.0000000 0.0000000 0.0000000 10.0000000 B
229  10.0000000 6.0000000 6.0000000 0.0000000 0.0000000 0.0000000 10.0000000 B
230  0.0000000 0.0000000 0.0000000 0.0000000 0.0000000 0.0000000 1.0000000 A
231  4.0000000 0.0000000 0.0000000 0.0000000 0.0000000 0.0000000 1.0000000 A
232  8.0000000 0.0000000 0.0000000 0.0000000 0.0000000 0.0000000 1.0000000 A
233  12.0000000 0.0000000 0.0000000 0.0000000 0.0000000 0.0000000 1.0000000 A
234 ```
235 
236 Первая строка - количество атомов.
237 
238 Вторая строка - размеры ячейки и названия столбцов.
239 
240 Ячейка задается тремя векторами (в общем случае ячейка триклинная). Пока поддерживается только прямоугольные ячейки. Размер по X - первое число, по Y - пятое, по Z - девятое. Остальные числа - нули.
241 
242 Типы столбцов используются следующие - Properties=pos:R:3:vel:R:3:mass:R:1:species:S:1. То есть три координаты, три проекции скорости, масса и имя атома (в таком порядке)
243 
244 С третьей строки идет информация об атомах.
245 
246 Для генерации ячеек написаны отдельные программы (https://github.com/AlexanderSidorenkov/xyz_create).
